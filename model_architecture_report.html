🔁 Pipeline Summary
FastFoundationStereo follows a classic stereo matching flow: feature extraction → cost volume → cost aggregation → iterative GRU refinement → upsampling. The key design choices are a 3D hourglass with per-scale image feature guidance for cost aggregation, and a Selective ConvGRU that blends small and large receptive fields based on a spatial attention map.
⛓
Block-by-Block Pipeline
Block 1
Feature Extraction
core/extractor.py · Feature
2 × (B, 3, H, W) → (B, 128, H/4, W/4) (B, 192, H/8, W/8) (B, 320, H/16, W/16) (B, 304, H/32, W/32)
🧠 EdgeNeXt-small backbone (pretrained) — 4-stage hierarchical encoder producing multi-scale features at strides 4, 8, 16, 32. Channel widths: 48 → 96 → 160 → 304.
🔺 FPN-style deconvolution — deconv32→16, deconv16→8, deconv8→4 with lateral skip connections. Final Conv1×1 projects to 128ch at stride-4 (includes ViT feature dim offset).
⚡ Efficiency trick — left and right images concatenated into one batch (2B, 3, H, W) for a single forward pass through the shared backbone, then split.
Estimated compute share
~10–15%
Block 2
Cost Volume Construction
core/submodule.py · build_gwc / build_concat
(B, 128, H/4, W/4) × 2 → (B, 32, D/4, H/4, W/4)
📐 GWC Volume (Group-wise Correlation) — features split into 8 groups, dot-product correlation for every disparity shift d=0..D/4-1. Shape: (B, 8, D/4, H/4, W/4). Optimised via PyTorch1 or Triton kernel.
🧩 Concat Volume — proj_cmb (Conv1×1: 128→12ch) projects each side, then left/right feature pairs concatenated for each shift. Shape: (B, 24, D/4, H/4, W/4).
🔗 Combined — channel-wise cat → (B, 32, 48, H/4, W/4). At 480×640 this tensor holds ~88M values — the single largest allocation.
Estimated compute share
~10–15%
Block 3
Cost Volume Stem
core/foundation_stereo.py · corr_stem + corr_feature_att
(B, 32, 48, H/4, W/4) → (B, 28, 48, H/4, W/4)
🔧 corr_stem — Conv3d 1×1: 32→28ch, then BasicConv 3D 3×3×3, then 2 × ResnetBasicBlock3D (28ch). Projects and refines the raw cost volume.
👁 FeatureAtt — squeeze-excite style: global-avg-pool the 2D image features (128ch), FC → scale factors applied to cost volume channels. Modulates cost volume by image content at stride-4.
Estimated compute share
~3%
Block 4 — ⚠ Bottleneck
Hourglass Cost Aggregation
core/foundation_stereo.py · hourglass
(B, 28, 48, H/4, W/4) → (B, 28, 48, H/4, W/4)
Input (28ch, D/4, H/4, W/4) │ ├─[Encoder]────────────────────────────────────────────── │ conv1: stride-2 3D + Conv3dNormActReduced → 56ch D/8 H/8 W/8 │ └─ FeatureAtt_8 (guided by x8 img feats, 192ch) │ conv2: stride-2 3D + Conv3dNormActReduced → 112ch D/16 H/16 W/16 │ └─ FeatureAtt_16 (guided by x16 img feats, 320ch) │ conv3: stride-2 3D + Conv3dNormActReduced → 168ch D/32 H/32 W/32 │ └─ FeatureAtt_32 (guided by x32 img feats, 304ch) │ ├─[Bottleneck Attention]───────────────────────────────── │ conv_patch: 4×4×4 strided → (28ch, D/64, H/64, W/64) │ CostVolumeDisparityAttention: │ 4-head Transformer × 4 layers (in disparity dimension) │ + sinusoidal positional embeddings │ interpolate ×4 → (28ch, D/4, H/4, W/4) [residual add] │ ├─[Decoder]────────────────────────────────────────────── │ conv3_up: deconv → 112ch │ cat with enc skip → 224ch → agg_0 (2×Conv3dNormActReduced) → 112ch │ └─ FeatureAtt_up_16 │ conv2_up: deconv → 56ch │ cat with enc skip → 112ch → agg_1 (2×Conv3dNormActReduced) → 56ch │ └─ FeatureAtt_up_8 │ conv1_up: deconv → 28ch │ + bottleneck residual │ conv_out: 2×Conv3dNormActReduced → 28ch │ Output (28ch, D/4, H/4, W/4)
Estimated compute share
~50–60% — dominant bottleneck due to 3D conv over 4D volume at multiple scales
Block 5
Classifier + Initial Disparity
core/foundation_stereo.py · classifier
(B, 28, 48, H/4, W/4) → (B, 1, H/4, W/4)
🎯 Classifier — BasicConv 3D: 28→14ch, ResnetBasicBlock3D, Conv3d 7×7×7: 14→1ch → logits (B, 48, H/4, W/4).
📉 Soft-argmax — softmax over disparity dim, weighted sum Σ p(d) · d → smooth initial disparity estimate (used as seed for GRU iterations).
Estimated compute share
~2%
Block 6
Context Network + Attention
core/extractor.py · ContextNetSharedBackbone
(B, 128, H/4, W/4) → net (128ch) + inp (128ch)
🧵 ContextNet — two parallel Conv2d 3×3 projections on x4: one for hidden state net (tanh), one for context inp (ReLU). Uses the same left features already computed in Block 1.
🔍 CAM (Channel Attention Enhancement) — squeeze-excite on inp. SAM (Spatial Attention Extractor) — produces spatial attention map used by SelectiveConvGRU. Both are very lightweight.
Estimated compute share
~1%
Block 7
Geometry Encoding Volume
core/geometry.py · Combined_Geo_Encoding_Volume
features_left/right[0] + cost vol → 2-level pyramid (sampled per iter)
🗺 Full correlation map — einsum dot-product between all left×right feature positions: (B, H/4, W/4, 1, W/4). Normalised L2. Built once.
🔭 2-level pyramid — cost volume and correlation map both avg-pooled in disparity dim × 2 for coarse/fine sampling. At each GRU iter, 1D bilinear sampling at disp ± corr_radius=4 → 9 samples × 2 levels × 2 sources = feature vector.
Estimated compute share (init + 8 lookups)
~5–8%
Block 8 — Iterative (×8 default)
Selective GRU Update Block
core/update.py · BasicSelectiveMultiUpdateBlock
net (128ch) + inp (128ch) + geo_feat → Δdisp (1ch) + mask (32ch)
🏃 BasicMotionEncoder — corr branch: Conv2d 1×1 + 3×3 → 256ch. Disp branch: Conv2d 7×7 + 3×3 → 64ch. Fuse: Conv2d 1×1 → 127ch + concat raw disp = 128ch motion features.
🔀 SelectiveConvGRU — blends two ConvGRU cells: small_gru (1×1 conv, narrow RF) and large_gru (3×3 conv, wider RF), mixed by spatial attention att. Adaptive receptive field per pixel.
📏 DispHead — Conv2d + ReLU, then 2 × EdgeNextConvEncoder (DWConv 7×7, expansion×4), Conv2d → Δdisp (1ch). Predicts the additive disparity update.
🧮 Mask head — 2 × Conv2d → 32ch. Produces per-pixel upsampling weights used in Block 9.
Estimated compute share (×8 iters total)
~10–15% total (cheap per iter, accumulates over iterations)
Block 9
Disparity Upsampling
core/foundation_stereo.py · upsample_disp
(B, 1, H/4, W/4) → (B, 1, H, W)
🔼 stem_2x — a 2-layer Conv2d branch (stem_2) produces 32ch features at H/2 × W/2 directly from the input image. Gives high-resolution texture cues.
⬆ Conv2x deconv (spx_2_gru) — upsample mask_feat_4 (32ch, H/4) with stem_2x (32ch, H/2) → 64ch at H/2. Then ConvTranspose2d → 9ch at full resolution.
🎨 context_upsample — the 9 channels are per-pixel soft weights for a 3×3 neighbourhood of the disparity map. Weighted sum produces sharp, edge-aware upsampled disparity at full H × W.
Estimated compute share
~1%

Optimisation Targets
Highest-impact targets for speed improvement
  • Block 4 — Hourglass (50–60%): Reduce number of hourglass levels, decrease volume_dim (default 28), reduce Transformer depth (num_transformer=4), use half-precision 3D convolutions, or replace 3D convs with 2D factorized convs.
  • Block 2 — Cost Volume (10–15%): Reduce max_disp or use the Triton-optimised kernel (optimize_build_volume='triton'). Lower cv_group (default 8) reduces GWC channels.
  • Block 8 — GRU Iterations (10–15%): Reduce valid_iters from 8 to 4–6. Quality vs. speed tradeoff. Hierarchical inference (run_hierachical) can help for high-res inputs.
  • Block 1 — Feature Extraction (10–15%): Switch to a smaller EdgeNeXt variant or prune the FPN deconv layers. Already shared between left/right via batch trick.
